만들어야 하는 페이지

1. 로그인 / 회원가입 페이지 (완료)

2. 송금하기 (홈 화면에서 알림 뜬 내용에 한 하여 완료)
    API상에서 입력 받는 거 : 상대계좌, 상대 잔액, 내 계좌, 내 잔액
    * 계좌 대 계좌 직접 송금 구현 필요
    1대1 송금하기를 위한 송금하기 탭을 만들고 그걸 네비게이션 바에도 추가해주고
    추가되었다면 alertcard 즉 홈에서 보이는 송금하기 버튼을 클릭할 경우
    해당 페이지로 이동하게 해줘야해.
    송금하기 기능 자체는 1대1 기반으로 사용자가 상대를 특정해서
    상대 계좌번호, 상대 거래은행, 거래금액을 입력하게 하고 송금 버튼을 누르면
    위에 토스트 느낌으로 거래 정보를 보여주고 정말 송금하시겠습니까?
    뜨면 예 누르면 송금이 완료되었습니다. toast 알림이 뜨고, 아니오 누르면 그냥 기존 송금하기 화면으로
    다시 돌아가게 해야해.
    
3. 마이페이지 (로그인 완료 시, 마이페이지로 연동까지만 구현)
    1. 내 계좌 등록 및 수정하기
    2. 정보 등록 및 수정하기
    * 마이페이지 구현 필요

    마이페이지 홈 구성 : 
    내 계좌 정보들(계좌번호, 은행, 잔액), 대표 계좌는 제일 상단에 위치한다.
    그 기준은 옆에 별표 클릭할 수 있게해서 별표가 있는 계좌가 대표 계좌여야함.
    그리고 이미 대표 계좌가 있을 때 빈 별표를 클릭해서 대표 계좌로 등록하려고 하면
    대표 계좌를 변경하시겠습니까? 라는 알림이 뜨고 예 아니오 중에 예를 선택하면
    대표 계좌가 변경되고 기존 대표 계좌는 그 아래 두번째 순서로 내려가도록 구성하면 좋을 것 같음.

    추가하기 버튼:
    계좌를 추가할 수 있는 버튼을 만들어서
    해당 버튼을 클릭하면 계좌 신규 등록 페이지로 넘어가서
    주요 정보들 있지? 뭐 계좌번호, 은행 이런 것들? 기입하고 등록하기 버튼을 누르면
    추가 완료 되었다는 알림과 함께 계좌 중에 제일 하단에 불러와질 수 있도록 하면 좋을 것 같음

    삭제하기 버튼 :
    삭제하기 버튼을 누르면 체크 표시를 할 수 있게 계좌별로 옆에 체크박스가 뜨고,
    체크박스로 1개 이상 클릭하여 체크를 하면 삭제하기 버튼이 활성화 되어서
    삭제하기 누르면 알림으로 계좌 정보 보여지면서 삭제 하시겠습니까? 뜨면
    예 아니오 중에 예 누르면 삭제되도록 하면 좋을듯. 

API는 추후에 명세서 줄테니까 우선 전체적인 UI 툴만 기존 UI 기반으로 일맥상통하게 해서 틀 만들어줘.


거래내역 탭
    완료된 거래만 보여줌
    1대1 송금 내역은 거래 상대, 거래 은행, 거래 금액, 거래 계좌가 보이게
    서류기반은 여기에 더보기 버튼 누르면 업로드했던 서류 보이게.   
    서류기반과 1대1송금 모두 포함하여 시간순으로 정렬해서 보여줌.(완료된 애들만 보임)
    서류기반 거래의 경우 → 더보기 버튼 클릭 시 사진을 보여줌.

서류보관함 탭
기존 보여주는 정보 + 사진 보여주는 것(업로드한 사진). 
(서류 업로드/ 서류 촬영하기 기반으로 입력된 서류만 보여줌)
완료된 거래는 보여주지 않음.
